## 一、审计概述

对 One API v0.6.10 进行安全审计，重点关注认证、授权、输入校验、数据传输、敏感信息保护等关键安全领域。

## 二、🔴 高风险问题

### 2.1 默认弱密码

**文件：** `common/const.go`

```go
RootPassword = "123456"
```

**问题描述：** 安装后不强制修改默认密码，攻击者可使用 `root/123456` 登录管理后台。

**风险等级：** 高

**建议：**
- 首次登录强制要求修改密码
- 或通过环境变量 `FORCE_ROOT_PASSWORD_CHANGE=true` 强制设置新密码
- 生产环境部署前务必修改默认密码

---

### 2.2 CORS 允许所有来源

**文件：** `middleware/cors.go`

```go
config.AllowAllOrigins = true
```

**问题描述：** 任何网站都可以跨域调用 API，可能导致 API 被恶意网站滥用。

**风险等级：** 高

**建议：**
```go
// 生产环境应配置白名单
config.AllowOrigins = []string{"https://yourdomain.com"}
// 或通过环境变量配置
config.AllowOrigins = strings.Split(os.Getenv("CORS_ALLOWED_ORIGINS"), ",")
```

---

### 2.3 Token 密钥硬编码

**文件：** `common/crypto.go`

```go
defaultSecretKey = "default secret key"
```

**问题描述：** 如未配置 `SessionSecret` 环境变量，使用已知默认密钥，可伪造会话 cookie。

**风险等级：** 高

**建议：**
- 启动时强制要求设置 `SessionSecret`，非可选配置
- 如检测到默认值，拒绝启动并提示配置

## 三、🟡 中风险问题

### 3.1 无 SSRF 防护

**文件：** `relay/channeltype/url.go`

```go
func GetFullURL(baseURL string, path string) string {
    // 只做简单拼接，无 URL 校验
}
```

**问题描述：** 用户可构造恶意 URL（如 `http://169.254.169.254/`）让服务端请求内网资源，获取云服务元数据。

**风险等级：** 中

**建议：**
```go
func GetFullURL(baseURL string, path string) string {
    // 添加 URL 校验
    parsed, err := url.Parse(baseURL)
    if err != nil {
        return ""
    }
    // 禁止内网 IP
    host := parsed.Hostname()
    if isPrivateIP(host) {
        return ""
    }
    // ...
}
```

---

### 3.2 无请求频率限制

**文件：** `middleware/rate_limit.go`

```go
func RateLimit() func(c *gin.Context) {
    return func(c *gin.Context) {
        // 仅作为占位符存在
        return c.Next()
    }
}
```

**问题描述：** API 无速率限制，可被滥用进行暴力破解或拒绝服务攻击。

**风险等级：** 中

**建议：** 实现真正的限流逻辑，可使用 Redis 作为计数存储。

---

### 3.3 日志泄露敏感信息

**文件：** `middleware/logger.go`

```go
fmt.Printf("OAuth2 token: %s\n", token)
fmt.Printf("Channel %d's key: %s\n", channel.ID, channel.Key)
```

**问题描述：** API Key、Token 等敏感信息会写入日志文件。

**风险等级：** 中

**建议：**
```go
// 脱敏处理
fmt.Printf("Channel %d's key: %s...\n", channel.ID, channel.Key[:8])
// 或使用结构化日志，降低泄露风险
log.Info("channel updated", "channel_id", channel.ID, "key_prefix", key[:8])
```

## 四、🟢 低风险/良好实践

### 4.1 密码使用 bcrypt

**文件：** `common/crypto.go`

```go
PasswordHash = bcrypt.DefaultCost
```

**👍** 密码存储使用 bcrypt，强度足够。

---

### 4.2 SQL 使用 GORM 参数化查询

**文件：** `model/user.go`

```go
DB.Where("id = ?", id).First(&user)
```

**👍** 使用参数化查询，无 SQL 注入风险。

---

### 4.3 RBAC 权限控制

**文件：** `middleware/auth.go`

```go
func authHelper(c *gin.Context, minRole int) {
    RequiredRole := c.GetInt("role")
    if RequiredRole > user.Role {
        // 权限不足
    }
}
```

**👍** 有基于角色的权限控制，支持普通用户、管理员、超级管理员三级。

---

### 4.4 Token 支持 IP 白名单

**文件：** `middleware/auth.go`

```go
if token.Subnet != nil && *token.Subnet != "" {
    if !network.IsIpInSubnets(ctx, c.ClientIP(), *token.Subnet) {
        abortWithMessage(c, http.StatusForbidden, "该令牌只能在指定网段使用")
    }
}
```

**👍** 支持 Token 绑定 IP 网段，增强安全性。

## 五、安全配置建议

### 5.1 生产环境必做

| 配置项 | 建议值 |
|--------|--------|
| `SessionSecret` | 使用随机字符串，32字节以上 |
| `RootPassword` | 强密码（12位，大小写+数字+特殊字符） |
| `AllowAllOrigins` | 改为白名单模式 |
| CORS | 仅允许受信任的域名 |
| 日志级别 | 避免写入敏感信息 |

### 5.2 推荐环境变量

```bash
# 认证配置
SessionSecret=your-random-secret-here-at-least-32-chars
FORCE_ROOT_PASSWORD_CHANGE=true

# CORS 配置
CORS_ALLOWED_ORIGINS=https://yourdomain.com,https://admin.yourdomain.com

# 数据库（如使用MySQL）
SQL_DSN="user:password@tcp(host:3306)/oneapi?charset=utf8mb4"

# Redis（如使用Redis）
REDIS_CONN_STRING=redis:6379
```

## 六、总结

| 风险等级 | 数量 | 主要问题 |
|----------|------|----------|
| 🔴 高风险 | 3 | 默认密码、CORS、密钥硬编码 |
| 🟡 中风险 | 3 | 无SSRF防护、无频率限制、日志泄露 |
| 🟢 低风险 | 4 | bcrypt、参数化查询、RBAC、IP白名单 |

**总体评价：** One API 代码质量较好，安全基础扎实，主要风险集中在默认配置和部分边界校验上。生产环境部署前按本文档建议进行配置，可有效降低安全风险。
